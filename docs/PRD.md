# 產品需求文件 (PRD) — 任務管理系統

## 1. 專案概述

- **專案名稱**：任務管理系統 (Task Management System)
- **專案描述**：這是一個基於 Web 的待辦事項管理工具，旨在協助使用者有條理地紀錄並追蹤日常任務，解決容易遺忘工作項目或缺乏工作焦點的問題。
- **背景與動機**：現代人生活與工作步調緊湊，經常有許多細碎的待辦事項。為避免遺漏重要事項並提升效率，需要一個簡單、直覺的任務管理系統來集中管理工作。
- **目標用戶**：需要管理日常工作、學習計畫或生活瑣事的學生與上班族。
- **核心價值主張**：以最少的點擊次數完成任務的記錄與狀態更新，提供「即寫即看、隨點隨清」的極簡體驗。

## 2. 功能需求

以下列出系統的主要功能，並以使用者故事 (User Story) 描述：

1. **新增任務 (Create Task)**
   - 作為 **使用者**，我希望 **能在輸入框輸入任務名稱並送出**，以便 **將新的待辦事項加入清單中**。
2. **查看任務清單 (Read Task List)**
   - 作為 **使用者**，我希望 **能看到目前所有的任務列表**，以便 **清楚知道還有哪些事情要做**。
3. **標記完成狀態 (Toggle Task Status)**
   - 作為 **使用者**，我希望 **能點擊任務旁邊的按鈕來切換完成或未完成狀態**，以便 **追蹤進度並獲得成就感**。
4. **編輯任務 (Update Task)**
   - 作為 **使用者**，我希望 **能點選某個任務進行內容修改**，以便 **在任務細節變更或打錯字時能夠更正**。
5. **刪除任務 (Delete Task)**
   - 作為 **使用者**，我希望 **能刪除已經不需要或不小心建立的任務**，以便 **保持清單的整潔**。
6. **篩選任務 (Filter Tasks)**
   - 作為 **使用者**，我希望 **能依照「全部」、「未完成」、「已完成」來篩選任務**，以便 **在不同情境下專注於特定狀態的事項**。

## 3. 非功能需求

- **技術限制**：
  - **後端框架**：使用 Python 的 Flask 框架處理 HTTP 請求與業務邏輯。
  - **模板引擎**：使用 Jinja2 在後端渲染 HTML 頁面，不採用前後端分離架構。
  - **資料庫**：使用 SQLite 儲存任務資料，確保輕量且無須額外設定資料庫伺服器。
- **效能與安全考量**：
  - 頁面加載速度應在 1 秒內。
  - 需防範基本的 XSS 攻擊 (Jinja2 預設會逸出 HTML 字元)。
  - 對使用者輸入進行基本長度限制，防止資料庫寫入過長的無意義字串。

## 4. MVP 範圍 (Minimum Viable Product)

- **Must Have (必須具備)**：
  - 顯示任務列表。
  - 新增任務。
  - 標記完成 / 取消完成。
  - 刪除任務。
- **Should Have (應該具備)**：
  - 依狀態篩選任務 (全部 / 未完成 / 已完成)。
  - 編輯現有任務的文字內容。
- **Nice to Have (未來擴充)**：
  - 設定任務的優先級 (高/中/低)。
  - 設定任務的截止日期 (Due Date)。
  - 任務的分類標籤 (Tags)。
  - 多使用者註冊與登入系統。
